## Overview

The application uses a combination of:
- **Global CSS** (`client/src/index.css`) - Base styles, design tokens, and common components
- **Component CSS** - Scoped styles for specific components
- **Inline Styles** - Dynamic styling based on data (tier colors, etc.)

## Best Practices

### Performance

1. **Use CSS Custom Properties for dynamic values**
```css
.player-row {
  border-left: 3px solid var(--tier-color, var(--line));
}
```
```jsx
style={{ "--tier-color": TIER_COLORS[tier] }}
```

2. **Will-change for animations**
```css
.staggered-menu-panel {
  will-change: transform;
}
```

3. **Contain for isolated layout**
```css
.staggered-menu-panel {
  contain: layout paint style;
}
```

### Accessibility

1. **Focus visible states**
```css
.sm-toggle:focus-visible {
  outline: 2px solid #ffffffaa;
  outline-offset: 4px;
}
```

2. **Hover states for interactive elements**
```css
button:hover:not(:disabled) {
  transform: translateY(-1px);
}
```

3. **Color contrast**
- Text on dark backgrounds: `#e8eaed` (light gray)
- Muted text: `#8892a8` (medium gray)
- Accent: `#5dade2` (bright blue for visibility)

### Maintainability

1. **Component-scoped CSS files**
```
StaggeredMenu.jsx
StaggeredMenu.css
```

2. **BEM-like naming conventions**
```css
.fc          /* Block */
.fc__slide   /* Element */
.fc__dot--active  /* Modifier */
```

3. **Design tokens in :root**
```css
:root {
  --accent: #5dade2;
  --shadow: 0 14px 30px rgba(0, 0, 0, 0.3);
}
```
