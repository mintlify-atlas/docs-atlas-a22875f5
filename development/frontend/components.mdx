## Overview

The frontend is built with React 19 and uses a component-based architecture. Components are located in `client/src/components/` and handle everything from navigation to animations.

## Core Components

### AppLayout

**Location**: `client/src/components/AppLayout.jsx`

The main layout wrapper that provides consistent navigation and structure across all pages.

```jsx
export default function AppLayout({ onTabChange, activeTab, children }) {
  const items = TABS.map((tab) => ({
    label: tab.label,
    link: "#",
    ariaLabel: tab.label,
    onClick: () => onTabChange(tab.id),
  }));

  return (
    <>
      <StaggeredMenu
        isFixed
        items={items}
        colors={["#2b6a4f", "#b6893f"]}
        accentColor="#b6893f"
        menuButtonColor="#e8eaed"
        openMenuButtonColor="#fff"
        displaySocials={false}
        displayItemNumbering
        logoAlt="Tullidos"
      />
      <div className="page-content">{children}</div>
      {activeTab !== "users" && <FooterCarousel />}
    </>
  );
}
```

**Props**:
- `onTabChange` - Function called when tab changes
- `activeTab` - Currently active tab ID
- `children` - Page content to render

**Features**:
- Manages tab navigation (Ranking, Usuarios, Informacion)
- Integrates StaggeredMenu for navigation
- Conditionally renders FooterCarousel
- Fixed positioning for menu

**Key Props**:

- `items` - Menu items with label, link, onClick
- `displayItemNumbering` - Show numbered items (default: true)
- `isFixed` - Fixed positioning (default: false)
- `menuButtonColor` - Button color when closed
- `openMenuButtonColor` - Button color when open
- `accentColor` - Accent color for highlights

**Features**:
- Layered slide-in animation with GSAP
- Staggered item entrance animations
- Animated plus icon that rotates to X
- Cycling "Menu"/"Close" text animation
- Click-away to close functionality
- Accessibility support (ARIA labels)

**Animation Timeline** (see `client/src/components/StaggeredMenu.jsx:72-127`):
1. Pre-layers slide in with stagger (0.055s between each)
2. Main panel slides in (0.5s duration, power3.out easing)
3. Menu items animate in (0.7s duration, staggered by 0.065s)
4. Item numbers fade in (0.45s duration)
5. Social links animate in if present

**Location**: `client/src/components/AnimatedList.jsx`

A scrollable list component with Motion (Framer Motion) animations and keyboard navigation.

```jsx

  items = [],
  renderItem,
  showGradients = true,
  enableArrowNavigation = true,
  displayScrollbar = true,
  initialSelectedIndex = -1,
  onItemSelect,
}) => {
  // Implementation with Motion animations
};
```

**Props**:
- `items` - Array of items to display
- `renderItem` - Function to render each item: `(item, index, isSelected) => JSX`
- `showGradients` - Display top/bottom fade gradients (default: true)
- `enableArrowNavigation` - Allow arrow key navigation (default: true)
- `displayScrollbar` - Show scrollbar (default: true)
- `initialSelectedIndex` - Initially selected item (default: -1)
- `onItemSelect` - Callback when item is selected

**Features**:
- Intersection Observer for scroll-triggered animations
- Fade-in animation on scroll into view
- Keyboard navigation (Arrow keys, Tab, Enter)
- Auto-scroll to selected item
- Gradient overlays for scroll indication
- Hover selection

**AnimatedItem Implementation** (see `client/src/components/AnimatedList.jsx:5-21`):
```jsx
const AnimatedItem = ({ children, delay = 0, index, onMouseEnter, onClick }) => {
  const ref = useRef(null);
  const inView = useInView(ref, { amount: 0.2, triggerOnce: false });
  return (
    <Motion.div
      ref={ref}
      initial={{ scale: 0.96, opacity: 0, y: 8 }}
      animate={inView ? { scale: 1, opacity: 1, y: 0 } : { scale: 0.96, opacity: 0, y: 8 }}
      transition={{ duration: 0.22, delay }}
    >
      {children}
    </Motion.div>
  );
};
```

## Best Practices

### Component Structure
- Keep components in `client/src/components/`
- Pair each component with its CSS file (e.g., `StaggeredMenu.jsx` + `StaggeredMenu.css`)
- Export as default for main components
- Use named exports for sub-components

### Props
- Provide sensible defaults for all props
- Document prop types in comments
- Use destructuring with defaults in function signatures

### Animation
- Use GSAP for complex timeline animations (StaggeredMenu)
- Use Motion for simpler entrance/exit animations (AnimatedList, FooterCarousel)
- Use Canvas + requestAnimationFrame for continuous effects (Squares)

### Performance
- Use `useCallback` for event handlers
- Use `useMemo` for expensive computations
- Clean up event listeners and animations in useEffect returns
- Use `will-change` CSS property for animated elements
