The backend implements a RESTful API using Express.js with a focus on caching, error handling, and graceful degradation.

## Application Setup

The server initializes with essential middleware and configuration:

```javascript
const app = express();

app.use(cors());
app.use(express.json());
app.use(express.static(path.join(__dirname, "public")));

const PORT = process.env.PORT || 3001;
```

**Location**: `server/index.js:9-15`

### CORS Configuration

CORS is enabled globally without restrictions to allow frontend access from any origin during development.

### Static File Serving

The server serves:
- Static assets from the `public` directory
- Built client application from `../client/dist` when available
- SPA fallback routing for non-API routes

## API Endpoints

### GET /api/player

Fetch ranked data for a specific player by Riot ID.

**Query Parameters**:
- `gameName` (string, default: "Azpy")
- `tagLine` (string, default: "1337")

**Response**:
```json
{
  "riotId": "Azpy#1337",
  "gameName": "Azpy",
  "tagLine": "1337",
  "puuid": "...",
  "profileIconId": 5379,
  "summonerLevel": 437,
  "soloq": {
    "tier": "DIAMOND",
    "rank": "II",
    "leaguePoints": 67

  "flex": null,
  "topChampions": ["Thresh", "Nautilus", "Leona"],
  "mainRole": "SUPPORT"
}
```

**Location**: `server/index.js:1061-1073`

### GET /api/ladder

Retrieve the complete ranked ladder with all tracked friends.

**Response**:
```json
{
  "players": [...],
  "cachedAt": "2026-03-13T10:30:00.000Z",
  "cacheTtlMs": 60000,
  "stale": false,
  "lastError": null,
  "ddragonVersion": "14.24.1"
}
```

**Behavior**:
- Returns cached data if available and fresh
- Triggers refresh if cache is empty
- Waits for ongoing refresh if one is in progress
- Falls back to stale cache on error with `stale: true`

**Location**: `server/index.js:1076-1125`

### GET /api/friends

```json
    "tagLine": "1337",
    "puuid": "...",
    "mote": "🎮"
  }
]
```

**Location**: `server/index.js:1128-1135`

### POST /api/friends

Add a new friend to the tracking list.

**Request Body**:
```json
{
  "gameName": "PlayerName",
  "tagLine": "EUW",
  "emote": "🏆"
}
```

**Response**:
- `201 Created` on success
- `400 Bad Request` if gameName or tagLine missing
- `409 Conflict` if player already tracked

**Side Effects**:
- Updates `friends.json`
- Triggers immediate ladder cache refresh

**Location**: `server/index.js:1138-1174`

### DELETE /api/friends/:gameName/:tagLine

Remove a friend from tracking.

**Response**:
- `200 OK` with `{ ok: true }`
- `404 Not Found` if player not in list
- `500 Internal Server Error` on file read failure

**Side Effects**:
- Updates `friends.json`
- Triggers immediate ladder cache refresh

**Location**: `server/index.js:1177-1204`

## Error Handling

### Validation Errors

Endpoints validate input and return appropriate 4xx status codes:

```javascript
if (!gameName || !tagLine) {
  return res.status(400).json({ error: "gameName y tagLine son obligatorios" });
}
```

### API Key Check

Endpoints requiring Riot API access check for key presence:

```javascript
if (!RIOT_API_KEY) {
  return res.status(500).json({ error: "Falta RIOT_API_KEY en server/.env" });
}
```

**Location**: `server/index.js:1062-1064`

### Graceful Degradation

The `/api/ladder` endpoint returns stale cached data when refresh fails:

```javascript
if (ladderCache.players.length > 0) {
  return res.json({
    players: playersWithEmotes,
    cachedAt: ladderCache.lastUpdatedAt,
    stale: true,
    lastError: err.message
  });
}
```

**Location**: `server/index.js:1112-1122`

### 404 Handler

Unknown routes return structured error:

```javascript
app.use((req, res) => {
  res.status(404).json({ error: `Ruta no encontrada: ${req.method} ${req.originalUrl}` });
});
```

**Location**: `server/index.js:1271-1273`

## Response Formats

### Success Response

Most endpoints return JSON with structured data:

```json
{
  "ok": true,
  "message": "Operation completed",
  "data": { ... }
}
```

### Error Response

Errors use consistent format:

```json
{
  "error": "Descriptive error message"
}
```

### Stale Data Indicator

Ladder responses include staleness flag:

```json
{
  "players": [...],
  "stale": true,
  "lastError": "Rate limit exceeded"
}
```

## SPA Routing

When client build exists, the server implements SPA fallback:

```javascript
app.get(/^(?!\/api\/|\/assets\/).*/, (req, res, next) => {
  if (req.method !== "GET") return next();
  return res.sendFile(CLIENT_INDEX_FILE);
});
```

**Behavior**:
- Serves `index.html` for non-API GET requests
- Preserves API routes under `/api/`
- Allows static assets under `/assets/`

**Location**: `server/index.js:1265-1268`

### Shutdown

Graceful shutdown handlers persist API stats:

```javascript
process.on("SIGINT", () => {
  saveApiStatsToFile(true);
  process.exit(0);
});

process.on("SIGTERM", () => {
  saveApiStatsToFile(true);
  process.exit(0);
});
```
