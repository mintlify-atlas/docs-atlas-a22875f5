## Quick Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Azpyrx/TullidosSoloqLadder.git
    cd TullidosSoloqLadder
    ```
  </Step>

  <Step title="Install dependencies">
    Install all dependencies with a single command from the root:
    
    ```bash
    npm install
    ```
    
    This automatically installs dependencies for both `server/` and `client/` via the postinstall script.
  </Step>

  <Step title="Configure environment variables">
    Create your environment file:
    
    ```bash
    cp server/.env.example server/.env
    ```
    
    Open `server/.env` and add your Riot API key:
    
    ```bash server/.env
    RIOT_API_KEY=RGAPI-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
    PORT=3001
    ```
  </Step>

  <Step title="Add friends to track">
    Edit `server/friends.json` to add players you want to track:
    
    ```json server/friends.json
    [
      {
        "gameName": "Azpy",
        "tagLine": "1337",
        "mote": "🎮"
      },
      {
        "gameName": "YourFriend",
        "tagLine": "EUW",
        "mote": "⭐"
      }
    ]
    ```
  </Step>

  <Step title="Start the backend">
    In one terminal, start the Express server:
    
    ```bash
    cd server
    npm start
    ```
    
    You should see:
    ```
    Server running on http://localhost:3001
    DDragon version: 14.24.1
    ```
  </Step>

  <Step title="Start the frontend">
    In another terminal, start the React development server:
    
    ```bash
    cd client
    npm run dev
    ```
    
    You should see:
    ```
    VITE v7.3.1  ready in 432 ms
    ➜  Local:   http://localhost:5173/
    ```
  </Step>

  <Step title="Access your ladder">
    Open your browser and navigate to:
    
    **http://localhost:5173**
    
    You should see your custom ladder with all tracked players sorted by rank!
  </Step>
</Steps>

## Verification

<Tabs>
  <Tab title="Backend Health">
    Check if the backend is running properly:
    
    ```bash
    curl http://localhost:3001/api/status
    ```
    
    You should see response with server statistics, rate limiting info, and daily highlights.
  </Tab>
  
  <Tab title="Frontend">
    The frontend should display:
    - Player cards sorted by rank
    - Rank badges for SoloQ and Flex
    - Daily activity highlights
    - Win rates and champion pools
    - Custom emotes for each player
  </Tab>
  
  <Tab title="Cache Files">
    After the first refresh, you should see these files created:
    
    ```bash
    ls server/
    ```
    
    - `ladder-cache.json` - Player data and rankings
    - `api-stats.json` - Request tracking and rate limit stats
  </Tab>
</Tabs>

## Troubleshooting

<Accordion title="Missing RIOT_API_KEY error">
  Make sure you've created `server/.env` and added your API key:
  
  ```bash
  RIOT_API_KEY=RGAPI-your-key-here
  ```
  
  Get your key from [developer.riotgames.com](https://developer.riotgames.com/).
</Accordion>

<Accordion title="Port already in use">
  If port 3001 or 5173 is already in use, change them:
  
  ```bash server/.env
  PORT=3002
  ```
  
  ```bash client/.env
  VITE_API_URL=http://localhost:3002
  ```
</Accordion>

<Accordion title="Player not found">
  Make sure the Riot ID is correct. The format is `GameName#TAG`:
  - GameName is case-sensitive
  - TAG is usually 3-5 characters (e.g., EUW, NA1, 1337)
  
  You can find your exact Riot ID in the League client or on op.gg.
</Accordion>

<Accordion title="Rate limit warnings">
  If you see rate limit warnings, the app will automatically:
  - Use cached data as fallback
  - Wait before retrying
  - Continue working with partial data
  
  This is normal behavior and ensures you stay within Riot API limits.
</Accordion>

## Development Tips

<Note>
The development server includes hot reload for both frontend and backend:
- **Frontend**: Vite automatically reloads on file changes
- **Backend**: Use `npm run dev` in server/ for hot reload with `--watch` flag
</Note>

<Tip>
Start with a small friends list (3-5 players) to test the app and understand rate limiting behavior before scaling up.
</Tip>
